Bandwidth-limited CubeSat downlinks motivate telemetry compression, but compression-ratio-first evaluation can be unsafe for spacecraft health data because small average reconstruction errors may hide channel-local excursions, limit violations, or anomaly signatures. This paper presents a safety-gated Edge-AI compression benchmark for CubeSat Electrical Power System telemetry using BIRDS on-orbit data. The benchmark evaluates lossless baselines, CCSDS-family proxy coding, fully connected and temporal learned autoencoders, and an all-numeric stress-test configuration. Instead of ranking methods only by compression ratio, each lossy model is screened using worst-case per-channel reconstruction-error gates derived from training data. All 32 lossless baseline rows pass the safety gate, whereas 149 EPS-only learned-compression rows and 143 all-numeric stress-test learned rows are rejected. To connect the benchmark with deployability, representative implementations are profiled across Raspberry Pi 4 ONNX Runtime inference, ESP32 lightweight compression, ESP32 TFLite Micro encoder-only inference, and STM32F429ZI/F413ZH MCU-class compression firmware. The study provides a pre-flight screening methodology for identifying compression methods that preserve spacecraft health visibility before mission-specific qualification.
